# GitHub Token 认证下 香港 vs 韩国 CDN 对比测试报告

## 测试概要

| 项目 | 内容 |
|---|---|
| 测试目标 | 对比 GitHub Token 认证下，香港与韩国 CDN 节点的并发承受能力 |
| 认证方式 | GitHub Personal Access Token (5,000 次/小时 API 限流) |
| 测试模式 | API 并发请求 + Git 并发 clone |
| 目标仓库 | `https://github.com/sgl-project/sglang.git` |
| 测试日期 | 2026-07-10 |
| 香港 CDN IP | `20.205.243.166` (github.com), `20.205.243.168` (api.github.com) |
| 韩国 CDN IP | `20.200.245.247` |

## 重要发现

### 韩国 IP 不支持 GitHub API

将 `api.github.com` 强制解析到韩国 IP `20.200.245.247` 后，**所有 API 请求均返回 404**。这表明：

- 韩国 IP 仅服务 `github.com`（git clone/push 协议），不承载 API 流量
- `api.github.com` 需单独路由，韩国 IP 上没有对应的 API 服务端点
- **如需使用 GitHub API，必须走香港或 API 专用节点**

| 请求类型 | 香港 IP (20.205.243.166) | 韩国 IP (20.200.245.247) |
|---|---|---|
| API 请求 | ✅ 正常 (200) | ❌ 全部 404 |
| Git clone | ✅ 正常 | ✅ 正常 |

## Token 认证 vs 无 Token 对比

| 对比维度 | 无 Token (前次测试) | Token 认证 (本次测试) |
|---|---|---|
| **香港首次 504 并发量** | 20 | 20 |
| **韩国首次 504 并发量** | 30 | **20** |
| **并发 5 平均 clone 耗时(香港)** | 7.99s | 7.60s |
| **并发 5 平均 clone 耗时(韩国)** | 6.06s | 6.98s |
| **并发 20 韩国 504 数** | 0 (0%) | **5 (25%)** |
| **并发 50 韩国 504 数** | 47 (94%) | **50 (100%)** |
| API 限流阈值 | 60 次/小时 | 5,000 次/小时 |
| API 并发 300 504 数 | N/A (被 403 拦截) | **0** |

**关键变化**：Token 认证下韩国节点的 504 馈线从并发 30 降到了并发 20，与香港持平。原因分析：

### Token 认证对 Git Clone 的影响

Token 嵌入 URL (`https://ghp_xxx@github.com/...`) 后：
1. **认证握手增加耗时** — 每次 clone 需完成 HTTP Basic Auth 验证
2. **SSL 连接开销** — Token 传输需要额外的 TLS round-trip
3. **韩国节点认证链路更长** — 韩国节点处理 Token 认证的延迟比香港高

这解释了韩国 clone 耗时从 6.06s→6.98s 的增加（+15%），以及 504 馈线从并发 30 退回并发 20。

## 综合结论

| 结论 | 说明 |
|---|---|
| Token 对 API 安全有益 | 香港 API 并发 300 仍零 504，大幅提升可用性 |
| Token 对 Git clone 有副作用 | 认证握手增加延迟，韩国 504 馈线从 30 退至 20 |
| 韩国 IP 不支持 API | `api.github.com` 必须走香港节点或独立路由 |
| 香港仍是 API 首选 | Token 认证 + 香港 CDN = 最高并发承受力 |
| 无 Token Git clone 韩国更优 | 韩国在无 Token 时并发承受力 > 香港 |

## 建议

| 使用场景 | 推荐方案 |
|---|---|
| GitHub API 调用 (CI/CD webhook 等) | Token + 香港 CDN，并发 ≤ 200 |
| Git clone/push (无 Token) | 韩国节点，并发 ≤ 20 |
| Git clone/push (有 Token) | 香港节点，并发 ≤ 14（韩国退至同等水平） |
| 企业代理配置 | API: `proxy_read_timeout` ≥ 10s; Git: ≥ 30s |
| 需同时用 API + Git | 香港 CDN（韩国不支持 API） |
